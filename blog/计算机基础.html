# 3. 常见的git指令

创建仓库/初始化/提交操作

  1. git init 初始化仓库操作，这样才能用git进行代码管理。
  2. git clone 复制远程仓库的代码到本地。
  3. git add XXX 添加本地的某个新文件到本地仓库，但是，此时只是提交到了本地仓库，并没有提交到远程仓库。
  4. git add . 这个操作和上面的区别在于，这个命令会添加所有的新文件，也就是当前目录下的。
  5. git commit -m 'message' 提交代码到本地仓库，并没有到远程仓库，不理解的可以去了解一下git的原理。
  6. git commit -am 'message' 这个命令将上面两个步骤 add 和 commit 合二为一。

日志查看/信息显示

  1. git log 这个命令主要用于查看提交日志
  2. git status 可以用来查看仓库的状态

分支管理

  1. 创建分支 git branch XXX 可以在远程界面创建分支，或者使用命令git branch XXX
  2. 切换分支 git checkout XXX
  3. 创建分支并且切换分支 git checkout -b XXX
  4. 查看处于哪个分支 git branch
  5. 查看本地和远程所有分支 git branch -a
  6. 合并分支 git merge
  7. 删除本地分支 git branch -D XXX
  8. 删除远程分支 git push origin --delete XXX

更新管理

  1. 提交代码到远程 git push origin XXX
  2. 拉取远程代码到本地 git pull origin XXX

版本回退

  1. git reset --hard XXX
